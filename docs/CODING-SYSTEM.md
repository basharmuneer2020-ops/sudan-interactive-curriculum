# 🏷️ نظام الترميز — دليل المساهمين

## الفلسفة

المنهج السوداني ضخم ومعقد — فيه مئات الدروس والتجارب التي لم نكتشفها بعد. لذلك نستخدم **نظام ترميز مرن** يسمح بـ:

- إضافة مهام مجهولة بأكواد محجوزة
- توسيع النظام بدون كسر الهيكل القائم
- تقسيم العمل بين المساهمين بسهولة
- اكتشاف الفجوات تلقائياً

---

## صيغة الكود

</div>

```
{STAGE}-{GRADE}-{SUBJECT}-{CHAPTER}-{TYPE}-{SEQ}
```

<div dir="rtl" align="right">

### 1. المرحلة (STAGE)

| الكود | المرحلة | الملاحظات |
|-------|---------|----------|
| `KG` | الروضة | صف واحد |

## كيف تستخدم النظام؟

### كمساهم:
1. **افتح جدول المهام** (`docs/task-matrix.xlsx`)
2. **ابحث عن كود** حالته "متاح" 🔴
3. **افتح Issue** على GitHub بعنوان: `[الكود] وصف المهمة`
4. **أنشئ فرع** باسم: `task/SEC-G11-CHE-C01-LAB-01`
5. **اعمل واقدّم** Pull Request

### كمنسق:
1. **أضف أكواد جديدة** عند اكتشاف دروس/تجارب جديدة
2. **حدّث الحالة** عند بدء أو اكتمال مهمة
3. **أضف أنواع جديدة** للـ TYPE عند اكتشاف آليات تطوير جديدة

---

## الحالات المتاحة

| الرمز | الحالة | المعنى |
|-------|--------|--------|
| 🔴 | متاح | جاهز للمساهمة — لم يبدأ أحد فيه |
| 🟡 | قيد العمل | مساهم يعمل عليه حالياً |
| 🟢 | مكتمل | تم تسليمه ومراجعته |
| ⚪ | محجوز | كود محجوز — المحتوى غير واضح بعد |
| 🔵 | مراجعة | مقدّم كـ PR وينتظر المراجعة |

---

## قابلية التوسع

النظام مصمم ليتوسع:

- **مادة جديدة؟** أضف كود من 3 حروف
- **نوع محتوى جديد؟** أضف كود من 3 حروف للـ TYPE
- **مرحلة دراسية جديدة؟** أضف كود المرحلة
- **اكتشفت 20 تجربة جديدة؟** أنشئ 20 كود بحالة ⚪ (محجوز)

**القاعدة الذهبية:** الكود يُنشأ فور اكتشاف المهمة، حتى لو التفاصيل غير واضحة.
